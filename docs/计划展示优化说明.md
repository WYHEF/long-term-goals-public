# 计划展示页面优化说明

## 优化内容

### 1. 布局优化 - 宽屏PC端设计

#### 原问题
- 内容全部竖向堆叠在中间狭窄区域
- 没有充分利用横向空间
- 阶段和周任务分离展示，缺乏关联性

#### 优化方案
- **容器宽度**：从 `max-w-4xl` 扩展到 `max-w-7xl`，充分利用宽屏空间
- **时间线总览**：采用左右分栏布局
  - 左侧（400px固定宽度）：显示开始日期、截止日期、计划周期
  - 右侧（flex-1）：阶段分布总览，2列网格展示
- **阶段详情**：采用左右分栏布局（4:8比例）
  - 左侧（33%）：阶段描述和目标
  - 右侧（67%）：周任务，2列网格展示

### 2. 层次化结构

#### 三级层次
```
阶段（可折叠）
  ├─ 阶段信息（左侧）
  │   ├─ 阶段描述
  │   └─ 阶段目标
  └─ 周任务（右侧，2列网格）
      ├─ 第1周
      │   ├─ 周重点
      │   └─ 每日任务（7天）
      ├─ 第2周
      └─ ...
```

#### 视觉层次
- 阶段头部：渐变背景 + 大号字体（text-lg）
- 周任务卡片：悬停高亮边框
- 每日任务：紧凑展示，减少垂直空间占用

### 3. 交互优化

- **阶段折叠**：点击阶段头部可展开/折叠，默认全部展开
- **快速导航**：点击时间线总览中的阶段卡片，自动滚动到对应阶段
- **内容编辑**：所有文字内容可直接点击编辑（保留原有功能）

### 4. 字体和间距优化

- 移除所有emoji表情符号
- 标题字体：text-base 到 text-lg
- 正文字体：text-sm 到 text-base
- 增加卡片间距，提升视觉呼吸感

## 技术实现

### 关键改动文件
- `src/views/goals/CreateGoalView.vue` - 主要优化文件
- `src/views/goals/PlanPreviewTest.vue` - 测试页面
- `src/router/index.js` - 添加测试页面路由

### 新增方法
- `setStageRef(el, index)` - 设置阶段DOM引用
- `scrollToStage(index)` - 滚动到指定阶段
- `getWeeksForStage(stage)` - 获取阶段内的周任务
- `getWeekIndex(weekNumber)` - 获取周任务索引

### 响应式设计
- 主要针对PC端优化（max-w-7xl）
- 使用grid布局实现左右分栏
- 周任务采用2列网格展示

## 使用建议

1. **开发调试**：使用测试页面快速预览效果，避免重复创建目标
2. **用户体验**：确保用户能清楚看到AI基于日期信息生成的计划
3. **内容编辑**：所有内容保持可编辑状态，方便用户调整
