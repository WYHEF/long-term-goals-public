# 管理员后台设置指南

## 🎯 功能概览

管理员后台提供以下功能模块：

- **📊 数据总览**：系统运营数据实时监控，包含统计卡片和快速操作
- **👥 用户管理**：查看和管理所有用户，支持数据导出
- **🎯 目标管理**：查看所有用户目标，支持类型/状态筛选和数据导出
- **✅ 打卡监控**：实时监控用户打卡情况，支持筛选和数据导出
- **📢 公告管理**：发布、编辑、删除系统公告（NEW！）
- **⚙️ 系统设置**：管理系统配置参数和功能开关（NEW！）

## 📦 安装步骤

### 1. 数据库配置

在 Supabase SQL Editor 中执行 `database/admin_schema.sql` 文件中的所有SQL语句。

**⚠️ 重要：用户管理功能需要额外配置**

为了正确显示用户邮箱和登录时间，需要在Supabase中创建一个数据库函数。

在 Supabase SQL Editor 中执行以下SQL：

```sql
-- 创建获取用户信息的函数
CREATE OR REPLACE FUNCTION get_users_info(user_ids uuid[])
RETURNS TABLE (
  id uuid,
  email varchar,
  created_at timestamptz,
  last_sign_in_at timestamptz
)
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = public
AS $$
BEGIN
  RETURN QUERY
  SELECT 
    au.id,
    au.email::varchar,
    au.created_at,
    au.last_sign_in_at
  FROM auth.users au
  WHERE au.id = ANY(user_ids);
END;
$$;

-- 授予执行权限
GRANT EXECUTE ON FUNCTION get_users_info(uuid[]) TO authenticated;
```

这个函数使用 `SECURITY DEFINER` 权限，允许前端通过RPC调用来获取用户的邮箱和登录信息。

这将创建：
- `admins` 表（管理员信息）
- `system_configs` 表（系统配置）
- `announcements` 表（系统公告）
- `admin_logs` 表（操作日志）
- RLS策略和权限控制
- `is_admin()` 函数用于权限检查

### 2. 添加管理员账号

执行以下SQL，将你的账号设置为管理员：

```sql
-- 首先找到你的用户ID
SELECT id, email FROM auth.users WHERE email = '你的邮箱@example.com';

-- 将用户添加为管理员
INSERT INTO admins (user_id, role, permissions) 
VALUES (
  '你的用户UUID',  -- 替换为上面查询到的ID
  'super_admin', 
  '["all"]'::jsonb
);
```

### 3. 验证权限

登录系统后：
1. 进入 "设置" 页面
2. 如果看到 "进入管理员后台" 按钮，说明配置成功
3. 点击按钮进入管理员后台

## 📊 核心功能详解

## 🔧 故障排除

### 问题1：看不到管理员入口

**解决方法**：
1. 检查数据库是否执行了 `admin_schema.sql`
2. 确认你的账号已添加到 `admins` 表
3. 查看浏览器控制台是否有错误

### 问题2：提示"没有管理员权限"

**解决方法**：
1. 检查 `admins` 表中的 `is_active` 字段是否为 `true`
2. 确认 `user_id` 与登录账号匹配
3. 重新登录系统

### 问题3：数据加载失败

**解决方法**：
1. 检查 RLS 策略是否正确配置
2. 确认 `is_admin()` 函数已创建
3. 查看 Supabase 日志

## 📥 数据导出功能

所有管理页面都支持数据导出功能：

**导出格式**：CSV（逗号分隔值）
**编码**：UTF-8 with BOM（兼容Excel）

**使用方法**：
1. 在任意管理页面（用户/目标/打卡）
2. 点击"📥 导出数据"或"📥 导出"按钮
3. 浏览器会自动下载CSV文件
4. 使用Excel或其他表格软件打开

**导出内容**：

- **用户数据**：用户ID、创建时间、主题、通知设置
- **目标数据**：目标ID、标题、类型、状态、进度、日期
- **打卡数据**：打卡ID、目标标题、日期、状态、用时、完成量

## 📝 使用建议

### 公告管理
1. **及时通知**：重要更新使用"警告"或"紧急"类型
2. **清晰简洁**：公告内容简明扼要
3. **及时清理**：过期公告及时停用或删除
4. **分类使用**：根据重要程度选择合适的类型

### 系统设置
1. **维护模式**：仅在必要时开启（如系统升级）
2. **注册控制**：根据系统负载适时关闭注册
3. **AI功能**：如API额度不足可临时关闭
4. **用户限制**：根据服务器性能调整目标数限制

### 数据导出
1. **定期备份**：建议每周导出一次关键数据
2. **大数据量**：数据量大时分批导出
3. **隐私保护**：导出的数据包含用户信息，注意保护隐私
4. **格式兼容**：CSV文件使用UTF-8编码，Excel可正常打开

## ⚠️ 注意事项

### 安全相关
1. **首次使用**：确保先执行数据库脚本
2. **权限控制**：谨慎分配管理员权限，只给可信任的人
3. **密码安全**：使用强密码保护管理员账号
4. **操作记录**：所有重要操作都会记录日志（开发中）

### 数据安全
1. **定期备份**：定期导出和备份数据库
2. **危险操作**：执行危险操作前务必确认
3. **二次验证**：删除和清空操作需要二次确认
4. **数据恢复**：删除的数据无法恢复，操作前务必谨慎

### 系统维护
1. **维护通知**：开启维护模式前先发布公告通知用户
2. **配置备份**：修改系统配置前记录原始值
3. **功能测试**：修改配置后及时测试功能是否正常
4. **监控日志**：定期检查系统运行状态和错误日志

## 🔧 常见问题

### Q1：公告不显示在用户端？
**A**：检查公告是否已启用（is_active = true），只有启用的公告用户才能看到。

### Q2：数据导出的CSV文件乱码？
**A**：使用Excel打开时选择"UTF-8"编码，或使用记事本打开后另存为。

### Q3：系统设置修改后不生效？
**A**：部分设置需要刷新页面或重新登录才能生效。

### Q4：如何查看某个用户的详细信息？
**A**：在用户管理页面点击"查看详情"（功能持续完善中）。

### Q5：误删了公告怎么办？
**A**：删除操作不可恢复，建议使用"停用"而不是"删除"。

### Q6：如何限制某个用户的操作？
**A**：目前暂不支持单个用户的权限控制，可以考虑在数据库层面操作。
