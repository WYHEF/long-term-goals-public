# 目标性质功能说明

## 功能概述

为了支持更灵活的目标管理，系统现在区分两种目标性质：

### 1. 🔄 持续性习惯
- **特点**：长期坚持，无固定结束时间
- **适用场景**：
  - 学习类：每日学英语、每天刷题、每天阅读等
  - 健康类：早睡早起、每天喝水、每天运动等
- **截止日期**：无需设置（`end_date` 为 `null`）

### 2. 🎯 阶段性目标
- **特点**：有明确的达成期限
- **适用场景**：
  - 学习类：考四级（6月15日）、考研（12月底）等
  - 健康类：减肥到60kg（3个月后）等
- **截止日期**：必须设置

## 数据库变更

### 新增字段

在 `goals` 表中新增字段：

```sql
goal_nature VARCHAR(20) DEFAULT 'staged'
```

可选值：
- `continuous` - 持续性习惯
- `staged` - 阶段性目标

### 迁移脚本

执行 `database/migrations/add_goal_nature.sql` 来添加此字段。

## 代码修改

### 1. CreateGoalView.vue

#### 新增表单字段
- 在步骤3中，为学习类和健康类都添加了"目标性质"选择
- 根据选择的性质，动态显示/隐藏截止日期字段

#### 验证逻辑更新
```javascript
const canProceedStep3 = computed(() => {
  // 必须选择目标性质
  if (!goalData.value.goalNature) return false
  
  // 阶段性目标必须填写截止日期
  if (goalData.value.goalNature === 'staged') {
    // 学习类检查 examDate
    // 减肥类检查 targetDate
  }
  
  // 持续性习惯无需截止日期
  return true
})
```

#### 创建目标逻辑更新
```javascript
// 根据目标性质决定截止日期
let endDate = null
if (goalData.value.goalNature === 'staged') {
  // 只有阶段性目标才设置 end_date
  endDate = userAnswers.value.examDate || healthConfig.value.targetDate
}
// 持续性习惯的 end_date 保持为 null
```

## 用户体验

### 创建目标流程

1. 输入目标名称和类型
2. AI分析（可跳过）
3. **选择目标性质** ⭐ 新增
   - 持续性习惯 or 阶段性目标
4. 填写其他详情
   - 如果选择"阶段性目标"，则显示截止日期字段（必填）
   - 如果选择"持续性习惯"，则隐藏截止日期字段
5. 生成并确认计划
6. 创建成功

### 示例场景

#### 场景1：持续性学习目标
- 目标：每日学英语30分钟
- 类型：学习类
- 性质：🔄 持续性习惯
- 截止日期：无
- 结果：目标可以一直进行，没有固定结束时间

#### 场景2：阶段性考试目标
- 目标：考英语四级
- 类型：学习类
- 性质：🎯 阶段性目标
- 截止日期：2025-06-15（必填）
- 结果：系统会根据截止日期规划学习进度

#### 场景3：持续性健康目标
- 目标：每天喝8杯水
- 类型：健康类 - 喝水
- 性质：🔄 持续性习惯
- 截止日期：无
- 结果：每天打卡，长期坚持

#### 场景4：阶段性减肥目标
- 目标：减肥到60kg
- 类型：健康类 - 减肥
- 性质：🎯 阶段性目标
- 截止日期：2025-08-01（必填）
- 结果：系统会根据目标体重和期限规划减肥计划

## 技术细节

### 前端验证
- 目标性质为必选项
- 阶段性目标必须填写截止日期
- 持续性习惯不显示截止日期字段

### 数据存储
- `goal_nature` 字段保存在 `goals` 表
- 持续性习惯的 `end_date` 为 `null`
- 阶段性目标的 `end_date` 不能为空

### 向后兼容
- 已有数据的 `goal_nature` 默认为 `staged`
- 迁移脚本会根据 `end_date` 是否为空自动设置 `goal_nature`

## 注意事项

1. **数据库迁移**：需要先在 Supabase 中执行迁移脚本
2. **已有数据**：已创建的目标会根据是否有 `end_date` 自动设置 `goal_nature`
3. **用户引导**：第一次使用时可能需要向用户说明两种目标性质的区别
4. **计划生成**：持续性习惯的计划生成逻辑可能需要后续优化（目前按周生成，可能需要支持更长周期）
