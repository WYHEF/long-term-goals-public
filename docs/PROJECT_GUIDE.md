# 长期目标管理网站 - 项目指南

## 📖 项目概述

这是一个基于 AI 辅助的长期目标管理和追踪系统，旨在通过即时反馈机制让长期目标的坚持变得更容易。

### 核心理念

**问题**：长期目标（学习、健身等）缺乏及时反馈，难以坚持
**解决方案**：通过 AI 辅助规划、可视化进度、打卡激励等方式提供即时正反馈

## 🎯 主要功能

### 1. AI 辅助目标创建

**流程**：
1. 用户输入目标（如"考英语四级"）
2. DeepSeek AI 分析并给出详细建议
3. 系统引导用户填写关键信息
4. AI 生成个性化计划
5. 用户确认并启动目标

**特点**：
- 详细的目标分析和建议
- 自动拆解为阶段性任务
- 精确到每日具体行动

### 2. 目标分类

#### 学习类
- 适用场景：考证、考试
- 特点：有明确截止日期
- 拆解方式：阶段目标 → 周任务 → 日任务

#### 健康类
- **睡眠**：记录睡眠时间，判断早睡和时长达标
- **喝水**：每日杯数统计
- **减肥**：体重记录和趋势分析
- **运动**：灵活记录各种运动项目

### 3. 灵活的严格程度

- **严格**：不可修改计划和截止时间
- **标准**：允许小幅调整
- **弹性**：可灵活修改各项参数

用户在创建目标时选择，系统会据此限制修改权限。

### 4. 打卡系统

#### 学习类打卡
- 完成情况（完成/部分完成/未完成）
- 实际完成量
- 用时记录
- 学习笔记

#### 健康类打卡
根据子类型有不同的表单：
- 睡眠：记录睡眠和起床时间
- 喝水：计数器式记录
- 减肥：体重、BMI、运动、饮食
- 运动：运动类型、量、时长

#### 打卡规则
- 当日任务：当天 00:00 - 次日 03:00 可打卡
- 缺卡后果：打断连续打卡、热力图空白
- 不允许补卡

### 5. 可视化进度

#### 进度仪表盘
- 总体进度百分比
- 已完成天数
- 剩余天数
- 连续打卡天数

#### GitHub 式热力图
- 全局热力图：显示所有目标的打卡
- 单目标热力图：特定目标的打卡历史
- 颜色深浅表示打卡频次

#### 进度条和图表
- 总体进度条
- 里程碑标记（25%、50%、75%、100%）
- 趋势分析图

### 6. 里程碑庆祝

**触发时机**：
- 进度达到 25%、50%、75%、100%
- 连续打卡 7 天、30 天、100 天
- 阶段目标完成

**展示方式**：页面内温和提示，不打断用户

### 7. 想法收集箱

**两种类型**：
- **新目标草稿**：记录突发想法，时机成熟时转为正式目标
- **资料链接**：收集与目标相关的学习资料

**特点**：
- 快捷输入框（全局可用）
- 可关联到现有目标
- 支持转换和删除

### 8. 数据统计

#### 总览数据
- 进行中目标数
- 已完成目标数
- 累计打卡次数
- 累计学习时长

#### 周报/月报
- 完成情况统计
- 学习时长趋势
- 最佳表现目标
- 鼓励性总结

## 🏗️ 技术架构

### 前端
- **框架**：Vue 3 + Composition API
- **构建工具**：Vite
- **UI**：Tailwind CSS（简约现代风格）
- **状态管理**：Pinia
- **路由**：Vue Router
- **图表**：ECharts（用于趋势图）

### 后端（BaaS）
- **服务**：Supabase
- **数据库**：PostgreSQL
- **认证**：Supabase Auth
- **实时同步**：Supabase Realtime

### AI 集成
- **模型**：DeepSeek API
- **用途**：目标分析、计划生成

### 数据安全
- **RLS**：Supabase 行级安全策略
- **认证**：基于 JWT 的用户认证
- **数据隔离**：用户只能访问自己的数据

## 🗄️ 数据库设计

### 核心表

#### goals（目标表）
- 基本信息：title, type, sub_type, strict_level, status
- AI 相关：ai_suggestion, user_answers
- 计划：plan (JSONB)
- 进度：progress (JSONB)
- 里程碑：milestones (JSONB)
- 健康配置：health_config (JSONB)

#### check_ins（打卡记录表）
- 基本信息：goal_id, date, status
- 学习数据：learning_data (JSONB)
- 健康数据：health_data (JSONB)
- 额外任务：extra_tasks

#### ideas（想法收集箱表）
- 类型：type（新目标/资料链接）
- 内容：title, description, link, notes
- 关联：related_goal_id
- 状态：status（pending/converted）

## 🔄 数据流

### 1. 创建目标
\`\`\`
用户输入 → AI分析 → 用户完善信息 → AI生成计划 → 用户确认 → 存储到数据库
\`\`\`

### 2. 打卡
\`\`\`
选择目标 → 填写打卡表单 → 提交 → 更新打卡记录 → 更新目标进度 → 检查里程碑
\`\`\`

### 3. 数据同步
\`\`\`
客户端操作 → Supabase实时同步 → 多设备自动更新
\`\`\`

## 📱 使用场景

### 场景1：考试备考
1. 创建目标"考英语四级"
2. AI 分析并建议3个月计划
3. 填写考试日期、当前水平
4. 获得每日学习任务
5. 每天打卡记录进度
6. 查看热力图保持连续

### 场景2：减肥计划
1. 创建目标"减肥10kg"
2. 填写当前体重和目标体重
3. 设置3个月期限
4. 每天记录体重
5. 查看体重曲线
6. 记录运动和饮食

### 场景3：养成习惯
1. 创建目标"早睡"
2. 设置弹性模式（可调整）
3. 每天记录睡眠时间
4. 系统自动判断达标
5. 累计连续天数
6. 热力图激励坚持
