# 持续性任务改进 - 更新日志

## 改进概述

为持续性任务（无固定结束日期的任务）添加了以下功能：
1. ✅ 可以查看之前的打卡记录
2. ✅ 每天的任务内容可以独立编辑
3. ✅ 可以在历史记录中查看和编辑过去的任务

## 详细改动

- **修改位置**: 第 108-277 行
- **改动内容**:
  - 将"每日计划时间线"改为仅在有 `end_date` 时显示（阶段性目标）
  - 为持续性任务（无 `end_date`）添加专门的"打卡历史记录"区域
  - 使用 `CheckInHistory` 组件展示历史记录

  - 添加任务的增删改功能
  - 实现保存功能，将修改同步到数据库

## 功能说明

### 持续性任务的特点
- 没有固定的结束日期（`end_date` 为 `null`）
- 使用灵活模式（`flexibleMode: true`）
- 每天的任务可以独立编辑

### 如何使用

#### 1. 查看打卡历史
1. 进入持续性任务的详情页面
2. 在"打卡历史记录"区域可以看到所有打卡记录
3. 点击任意记录可以展开查看详情

#### 2. 编辑历史任务
1. 展开某条打卡记录
2. 在"今日任务"区域点击"✏️ 编辑"按钮
3. 可以修改任务内容、添加新任务或删除任务
4. 点击"✅ 保存"按钮保存修改

#### 3. 每日打卡时编辑任务
1. 在打卡页面（CheckInView）
2. 如果是持续性任务，会显示"编辑任务"按钮
3. 可以基于昨天的任务进行修改
4. 打卡时会保存当天的任务列表

## 数据结构

打卡记录中的任务数据存储在 `learning_data.todayTasks` 字段：
```json
{
  "learning_data": {
    "todayTasks": [
      "完成Python教程第3章",
      "练习算法题5道",
      "复习昨天的内容"
    ],
    "timeSpent": 120,
    "notes": "今天学习效率很高",
    ...
  }
}
```

## 兼容性说明

- ✅ 不影响阶段性目标（有 `end_date` 的目标）的功能
- ✅ 不影响健康类目标的打卡
- ✅ 向后兼容旧的打卡记录（没有 `todayTasks` 的记录）

## 测试建议

1. **创建持续性任务**
   - 创建一个学习类目标
   - 选择"持续性习惯"
   - 不设置结束日期

2. **测试打卡功能**
   - 进行第一次打卡，编辑今日任务
   - 进行第二次打卡，查看是否复制了昨天的任务
   - 修改任务内容后打卡

3. **测试历史记录**
   - 查看目标详情页的"打卡历史记录"
   - 展开某条记录，查看任务列表
   - 编辑历史任务，保存后刷新页面验证

4. **测试编辑功能**
   - 添加新任务
   - 删除任务（至少保留一个）
   - 修改任务内容
   - 保存并验证
